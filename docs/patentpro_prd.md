版本：v1.0（整合版）  
更新时间：2026-03-30  
适用范围：官网首页、服务页、样例页、FAQ、提交项目页

---

## 1. 项目背景与目标

本 PRD 用于整合以下两类输入，形成一份可直接进入设计与开发的统一文档：

- `docs/patentpro_website.md`：官网定位、信息架构、模块职责、命名与设计要求
- `docs/patentpro_context.md`：首页终稿级文案与逐屏表达

核心目标：

- 保持品牌与定位统一，避免“平台腔、AI 腔、代理机构腔”
- 让官网信息从“能看懂”升级到“能转化”
- 明确“前置预判”与“正式法律意见”边界，避免误导

## 2. 品牌与定位（统一口径）

### 2.1 品牌定位

- 品牌名：**知域科技**
- 产品方向：**AI 驱动的智能专利预判**

### 2.2 官网一句话定位（主表达）

**让专利判断更早、更快、更省钱**

### 2.3 官网副定位（解释型）

**面向中小型科研机构和科技企业，用 AI 把技术梳理、查新判断和 FTO 风险筛查前置到立项、研发和产品落地更早阶段**

### 2.4 价值定义（全站底层逻辑）

本产品卖的不是“检索动作”本身，而是：

**在正式申请、正式 FTO、正式撰写之前，先用 AI 做一轮低成本、高速度、可继续往下走的专利判断。**

---

## 3. 文案策略与禁区

### 3.1 文案风格

- 说人话，像老板开会时会说的话
- 结果导向，不讲抽象能力堆砌
- 有 AI 概念，但不使用 AI 腔表达
- 短句、可扫读、可决策

### 3.2 禁区表达（避免作为首页核心文案）

- 专利前置分析服务
- 多 Agent 协同
- 智能编排 / 自主推理 / 结构化引擎
- 知识产权基础设施 / 全流程闭环 / 降本增效解决方案

### 3.3 推荐高频核心词

- 智能专利预判
- AI 驱动
- 更早判断
- 更低成本
- 先筛一轮
- 先看清再投入
- 把重投入后置
- 把高风险前置暴露

## 5. 首页 PRD（最终整合结构）

为兼容“7 屏策略”和“终稿文案完整性”，首页采用 **7 个核心模块 + 5 个增强模块**：

- 核心模块用于主流程转化（必做）
- 增强模块用于减少顾虑、提升信任（建议做）

### 5.1 Header（增强模块）

- Logo：**知域科技**
- 导航：首页 / 技术交底梳理 / 查新判断 / FTO 风险筛查 / 样例结果 / FAQ
- 右上 CTA：**提交项目**

### 5.2 模块 1：首屏 Hero（核心）

- 品牌上方文案：**AI 驱动的智能专利预判**
- 主标题：**让专利判断更早、更快、更省钱**
- 副标题：  
  **面向科研机构和科技企业  
  先做技术梳理、查新判断和 FTO 风险筛查  
  先看清，再投入**
- 按钮：**提交项目** / **查看样例**
- 辅助卖点：**无需自购专利数据库** / **低成本启动预判** / **可选人工复核**

### 5.3 模块 2：为什么现在就要做预判（核心）

- 标题：  
  **很多专利问题  
  不是做不了  
  而是做得太晚**
- 卡片 1：**申请前不判断，钱容易花错**
- 卡片 2：**上市前才看 FTO，风险往往已经太晚**
- 卡片 3：**材料没理顺就往下走，后面只会更贵**

### 5.4 模块 3：你能做什么（核心）

- 标题：**先把最关键的 3 件事看清楚**
- 服务 1：**技术交底梳理**（先把技术讲清楚）
- 服务 2：**查新判断**（先看值不值得申请）
- 服务 3：**FTO 风险筛查**（先筛一轮明显风险）
- 服务卡 CTA：统一为 **查看详情**

### 5.5 模块 4：为什么比传统方式更省钱（核心）

- 标题：  
  **不是所有项目  
  都值得一上来就重投入**
- 副标题：  
  **先做低成本预判，再决定要不要做正式查新、正式 FTO、正式撰写**
- 卖点 1：**不用先买数据库**
- 卖点 2：**让 FTO 可以更早开始**
- 卖点 3：**把重投入后置**

### 5.6 模块 5：你最终拿到什么（核心）

- 标题：  
  **不是几句回复  
  而是一份判断结果**
- 副标题：**重点回答 4 个问题**
  - 值不值得做
  - 核心点在哪里
  - 风险点在哪里
  - 下一步做什么
- 结果栏目建议：技术问题 / 核心方案 / 关键创新点 / 接近风险 / 信息缺口 / 下一步建议
- 按钮：**查看样例结果**

### 5.7 模块 6：适合谁（核心）

- 标题：**这类团队最需要先做预判**
- 人群卡片：
  - 科研机构负责人
  - 科技公司老板
  - 研发负责人
  - IP / 法务 / 成果转化负责人

### 5.8 模块 7：底部 CTA（核心）

- 标题：  
  **手里有项目  
  现在就可以先看第一轮**
- 副标题：  
  **不用先把材料写完整  
  也不用先买数据库**
- 按钮：**提交项目** / **预约沟通**

### 5.9 合作流程（增强模块）

- 标题：**先把材料发过来，我们先看第一轮**
- 四步：
  1. 提交材料
  2. 先做梳理和补问
  3. 生成预判结果
  4. 再决定后续动作

### 5.10 信任与边界（增强模块）

- 左侧（价值）：先讲清技术、先筛明显风险、先判断投入价值
- 右侧（边界）：
  - 不承诺授权结果
  - FTO 风险筛查不等于正式 FTO 法律意见
  - 结果不能替代最终申请文件
  - 正式法律与申请工作仍由专业代理人/律师完成

### 5.11 样例区（增强模块）

- 标题：**先看一份样例，比听概念更直接**
- 按钮：
  - 看技术交底梳理样例
  - 看查新判断样例
  - 看 FTO 风险筛查样例

### 5.12 FAQ 摘要区与 Footer（增强模块）

- FAQ 摘要保留 4 个高频问题：机构区别、为什么省钱、数据库是否必须、结果是否可直接申请
- Footer 固定声明：  
  **本服务用于专利相关前置判断与风险筛查，不构成正式法律意见**

## 6. 服务页 PRD（命名 + 文案）

### 6.1 服务命名（对外）

- 技术交底增强 -> **技术交底梳理**
- 查新预分析 -> **查新判断**
- FTO 风险初筛 -> **FTO 风险筛查**

说明：内部流程命名可保留原专业术语，对外统一采用上述结果导向命名。

### 6.2 技术交底梳理页

- 主标题：**技术总是写不清，先把材料理顺**
- 副标题：把零散说明、PPT、图纸和想法整理成可继续推进的技术说明
- 你会得到：技术核心点梳理 / 创新点提炼 / 信息缺口提示 / 下一步建议
- CTA：**提交技术交底材料**

### 6.3 查新判断页

- 主标题：**先看清像不像别人做过，再决定值不值得申请**
- 副标题：先判断和现有公开方案有多近，是否有继续申请空间
- 你会得到：接近方向梳理 / 主要风险提示 / 申请空间判断 / 下一步建议
- CTA：**提交查新判断项目**

### 6.4 FTO 风险筛查页

- 主标题：**产品往前走之前，先筛一轮专利风险**
- 副标题：适合上市、量产、合作、出海前，先看重点深查方向
- 你会得到：高风险模块提示 / 重点深查方向 / 是否建议正式 FTO / 下一步建议
- 边界高亮：**这是前置风险筛查，不是正式法律意见**
- CTA：**提交 FTO 风险筛查项目**

## 7. 提交项目页 PRD

### 7.1 页面文案

- 主标题：**先把材料发过来，我们先看第一轮**
- 副标题：方案说明、PPT、图纸、论文、旧专利、竞品资料都可以作为起点

### 7.2 表单字段

1. 你现在最想先看什么（讲清技术 / 值不值得申请 / 有无明显风险 / 先沟通）  
2. 项目现在走到哪一步（想法 / 研发 / 申请准备 / 落地或合作或出海前）  
3. 这项技术是做什么的（自由文本）  
4. 你现在最担心什么（接近风险 / 护城河不足 / 材料不清 / 侵权风险 / 投入过重）  
5. 上传现有材料（说明：不要求完整，有什么先传什么）  
6. 你希望怎么开始（直接开始预判 / 先沟通一次）  
7. 按钮：**提交材料**

## 8. FAQ（完整版）

建议保留以下 6 题：

1. 你们和专利代理机构有什么区别  
2. 为什么说这个方式更省钱  
3. 不买专利数据库，也能做吗  
4. FTO 风险筛查是不是正式 FTO  
5. 结果能直接拿去申请吗  
6. 材料不完整，也能做吗

每题回答统一结构：

- 先给结论（能/不能/不是）
- 再解释前置预判价值
- 最后补充边界声明（正式法律与申请仍需专业主体）

## 9. 设计与排版规范

### 9.1 视觉气质

- 专业、清晰、短句、结果导向
- 不做聊天产品感
- 不做传统律所感
- 不做数据库工具感

### 9.2 品牌呈现

- 首屏优先展示 **知域科技**
- “AI 驱动的智能专利预判”作为紧随其后的产品定位
- 英文名或域名仅辅助展示，不抢主视觉

### 9.3 排版约束

- 标题尽量双行内解决
- 副标题控制在 2-3 行
- 卡片标题优先 8 字以内
- 卡片说明优先 2-3 行
- 标题/按钮/卡片句默认不带句号

## 11. 冲突消解与合并说明

本次整合已处理以下潜在冲突：

1. **首页屏数冲突**  
   - 结构稿提出“7 屏核心化”，终稿文案包含更多内容。  
   - 已统一为“7 核心 + 5 增强”，兼顾转化效率与说服力。

2. **副标题长短冲突**  
   - 已采用“首屏短副标题 + 其他模块补充解释”的分层方式。

3. **服务命名冲突**  
   - 统一采用对外结果导向命名：技术交底梳理 / 查新判断 / FTO 风险筛查。

4. **能力与边界冲突风险**  
   - 在首页、服务页、Footer、FAQ 中统一增加边界声明，防止被理解为正式法律意见。
